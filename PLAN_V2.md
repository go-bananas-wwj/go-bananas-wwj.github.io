## 一、首屏 Hero 重构（优先级：高）

### 当前问题
- 头像太小，文字全部居中堆叠，缺乏视觉层次

### 改进方案
- **布局**：左图右文，图片占 40%，文字占 60%
- **图片**：放大至 200-240px，增加 subtle 边框或 shadow
- **右侧内容**：
  - 大号姓名（48px+）
  - 一行简洁定位语
  - 2-3 句核心介绍
  - 社交链接图标行（GitHub、ModelScope、Email）
- **背景**：增加极淡的渐变或几何装饰，打破纯白的单调

### 参考
- Brittany Chiang / Lee Robinson 的 Hero 区：大图 + 简洁文字 + 充足留白

## 二、首页内容重组（优先级：高）

### 删除
- 本科创业经历的详细描述（移至简历页即可）
- "关注 Vibe Coding / Agent 架构前沿" 等偏技术表述

- **爱好/日常区**（替换原 About）：
  - 简短描述平时爱搞点小东西（硬件、智能体、小工具）
  - 用图标 + 简短标签的形式展示（如 🔧 硬件 tinkering、🤖 Agent 实验、🗺️ 地理数据可视化）
- **最新动态**：可选展示最近的一篇随记/文章摘要

## 三、设计风格升级（优先级：高）

### 当前问题
- "太素了"——全白背景 + 淡蓝点缀，缺乏记忆点

### 改进方向
1. **增加深色模式切换**（🌙/☀️ 按钮）——高 Star 主页的标配
2. **背景层次**：
   - 首页 Hero 区增加极淡的渐变背景（淡蓝到白的微妙过渡）
   - 或增加 subtle 的网格/点阵装饰背景
3. **字体层次升级**：
   - 标题用更粗的 weight + 更大的字号对比
   - 引入 letter-spacing 和 line-height 的精细调整
4. **卡片风格**：
   - 增加 hover 时的 subtle 阴影和位移
   - 边框用极淡的颜色，hover 时显现
5. **动画升级**：
   - 页面加载时的 stagger 入场动画（元素依次淡入）
   - 鼠标跟随的 subtle 光效（可选）

### 参考风格
- **Brittany Chiang**：深色主题 + 高对比文字 + 精致排版
- **Lee Robinson**：大量留白 + 大字体 + 内容为王
- **Anthony Fu**：极简 + 内容密度低 + 笔记/博客驱动

## 五、新增「随记」页面（优先级：高）

### 功能需求
- 新增导航 tab：首页 / 简历 / 随记
- 支持 Markdown 格式的文章/笔记
- 文章列表页：标题 + 日期 + 摘要
- 文章详情页：完整 Markdown 渲染
- 用户可以每天更新内容

### 技术方案
- 文章数据存储在 `src/data/notes/` 目录下的 Markdown 文件
- 每篇文章一个 `.md` 文件，Frontmatter 包含标题、日期、标签
- 用 `marked` 库在客户端渲染 Markdown
- 文章列表按日期倒序排列

### 用户操作
- 更新文章：在 `src/data/notes/` 下新建或编辑 `.md` 文件
- 提交后 GitHub Actions 自动重新部署
